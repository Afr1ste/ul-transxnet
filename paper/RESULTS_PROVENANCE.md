If a revision requires a strict clean-only provenance trail for every baseline, rerun the full baseline pool on the rebuilt BUSI/AUL directories.

## Submission-readiness notes

- Main results are mean +/- standard deviation across three TN5000 seeds or five BUSI/AUL folds.
- Automatic ROI tables are generated from detector-predicted VOC annotations and reuse the existing GGG-MCA classifier checkpoints; the classifier is not retrained for the automatic/full-image comparisons.
- Bootstrap confidence intervals and calibration metrics are generated from existing prediction CSVs using `generate_paper_statistical_calibration.py`; case-level diagnostic statistics and reliability curves are generated from frozen prediction CSVs using `generate_high_roi_no_retrain_tables.py`.
- Android latency includes ROI preprocessing and ONNX Runtime inference, not only model-only CUDA timing.
- MCA is described as improving cross-dataset operating behavior, not as a guaranteed AUC-improving switch.
- The 2026-05-05 no-retrain revision pass used only frozen completed-result CSVs and existing automatic-ROI summaries. It did not reload current intermediate datasets, manifests, checkpoints, or images because the working dataset folders may have label drift relative to the frozen logs.
